Classify Snoring using in RNN and LSTM on TIHM Dementia Data

0

In the United States, there are nearly seven million prople living with Alzheimer's Disease. The cost of care is nearly $360 billion.

The TIHM project originates in England and is an open dataset for remote healthcare monitoring in dementia patients. It monitors patients in their homes using low-cost Internet of Things (IoT) sensing technologies. The sensing technologies collect vital signs and sleep patterns of patients in various rooms of their homes. The patients had to be 50 years old with a verified diagnosis of dementia. There was sleep data for 17 participants. On average each participant was involved in the study for about 50 days.

The data is time-series data, so I used an Recurrent Neural Network (RNN) and LSTM to classify patients as snoring or not. I am interested in snoring information because there have been interest in snoring or not snoring data for patients with dementia. The results of the analysis can be found in the following workbooks: RNN Analysis Workbook LSTM Analysis Workbook Sample Analysis Workbook

Dataset Overview

The sleep data in the TIHM project is a large dataset with 461,423 cases and 6 variables. I samples 1% of the data resulting in 4,614 cases. I selected two feautres of interest - heart rate and respiratory rate. The sample had coverage of the population because the average of the features were similar in both the sample and the population.

Data Analysis: RNN Model

The RNN model had a hidden size of 16 and used binary classification to classify patients as snoring or not. I trained on 100 epochs and calculated the loss values and the accuracy values during training. My model has a tendency to overfit, so I included a drop out rate of 0.4. The drop out rate means that all the neurons are not used to train the model.

The Loss function declines from 0.55 to 0.0068 by the hundredth epoch. This is a great loss value as it is close to 0.

The accuracy during training remains at 100%. Accuracy is the degree of correctness of the result. The model is highly accurate, despite only having two features. The model is most likely overfitting because there is not enough training data.

Evaluate the Model

The test set has a test loss of 0.3 and a test accuracy of 100%. Both of these values are good as the loss is close to zero and the accuracy is close to 100%.

Data Analysis: LSTM Model

The LSTM model is trained on the same sample of data as the RNN model. It has two drop outs set at 0.2. It is trained for 20 epochs.

The Loss function declines from 0.17 to 5e-6 by the twentieth epoch. This is a great loss value as it is close to 0 and only takes 20 epochs.

The accuracy during training remains at 100%. Accuracy is the degree of correctness of the result. The model is most likely overfitting because there is not enough training data in the sample. It is only trained on two features, so increasing the complexity of the data may result in less overfitting.

Evaluate the Model

The test set has a test loss of 2.94e-06 and a test accuracy of 100%. Both of these values are good as the loss is close to zero and the accuracy is close to 100%.
